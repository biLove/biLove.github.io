JavaScript 循环、函数、数组

2017.12.18 Lilya Bao 编程

编码的核心是：解决问题的方法和为了达到目的所采取的步骤。

什么是算法？
算法即解决问题的方法。

什么是流程图？
流程图是一种图表，通过一系列的逻辑语句概述了问题的解决方案。这些语句的评估和执行顺序叫做控制流程。

流程图可以帮忙梳理问题的逻辑，使用流程图，可以直观的看到算法的逻辑，防止遗漏细节。

条件语句

常见的条件语句是：if…else

关于 if…else 语句的注意事项。

  1. if 语句中的值始终转换为 true 或 false。根据该值，if 语句中的代码会运行，或 else 语句中的代码会运行。
  2. if 和 else 语句中的代码位于花括号 {…} 里，以便区分条件，并表明要运行哪段代码。
  3. 可能问题提示：编程时，有时候你可能只想使用 if 语句。但是，如果你尝试只使用 else 语句，就会出现错误 SyntaxError: Unexpected token else。出现此错误是因为 else 语句需要 if 语句才能运行。没有 if 语句的话，则不能使用 else语句。

逻辑运算符

逻辑表达式类似于数学表达式，但是逻辑表达式的结果是 true 或 false。

&& 符号是逻辑 AND 运算符，用于将两个逻辑表达式结合为一个更大的逻辑表达式。
如果两个更小的表达式为 true，那么整个表达式结果为 true。
如果其中一个更小的表达式为 false，那么整个逻辑表达式为 false。

运算符有以下三种：

  1. && —— 逻辑AND，示例： value1 && value2
  2. || —— 逻辑OR，示例： value1 || value2
  3. ！ —— 逻辑NOT，示例： !value1，返回value1的相反值，如果value1为true，则!value1为false

真值和假值

结果为 false 的值称为 假值。

例如，空字符串 “” 为假值，因为在布尔表达式中，”” 等于 false。

false == “”
返回：true

所有假值的列表：

  1. false
  2. null
  3. undefined
  4. 0
  5. NaN
  6. “”

本质上，如果不是假值，则为真值！

Switch 语句

如果你的代码中重复出现 else if，而且每个条件都是基于相同的值，那么很大可能可以使用 switch 语句。

switch 语句是另一种将多个基于相同值的 else if 语句放到一起，并且不用使用条件语句的方式，只是根据某个值切换每段代码。

如果不使用中断语句，switch 会一直运行直到结束。

中断语句

中断语句可以用来终结一个 switch 语句并将控制权转给被终结语句后面的代码。向每个 case 条件添加 break 后，就解决了 switch 语句不断往下跳转到其他 case 条件的问题。

default

可以再 switch 语句中添加 default 条件，当没有任何与 switch 表达式相符的值时，将执行该语句。

函数

函数的作用是，将相同的步骤打包成可以重复使用的代码块。格式如下：（关键词，函数名，参数）

为了让函数执行任务，我们需要调用函数，方法是使用函数名称，后面是小括号，其中包含任何传入的参数。

函数的返回值

所有的函数都会给调用者返回某个值，如果没有指定返回值，则函数将自动返回 undefined

返回值有两种方法：

  1. 使用 console.log(value) 可以将value 的值输出到控制台。
  2. 使用关键字 return 和一个值

return

  1. return 用在函数停止执行的地方，如果连续出现多个return，则第一个生效，后面的无效；
  2. return 会返回一个值给调用者，如果没有定义返回值，则会自动返回 undefined

总结

  1. 如果标识符在全局作用域内声明，则可以到处访问。
  2. 如果标识符在函数作用域内声明，则可以在所声明的函数内访问（甚至可以在函数中声明的函数内访问）。
  3. 尝试访问标识符时，JavaScript 引擎将首先查看当前函数。如果没找到任何内容，则继续查看上一级外部函数，看看能否找到该标识符。将继续这么寻找，直到到达全局作用域。
  4. 全局作用域不是很好的做法，可能会导致糟糕的变量名称，产生冲突的变量名称和很乱的代码。

提升 hoisting

有时候，JavaScript 代码可能会产生第一眼看着不明显的错误。提升可能是某个导致很难调试的错误的罪魁祸首。
正常情况下， JavaScript 解析代码会按照顺序，但是 JavaScript 有提升的功能。
例如，有一个变量的调用在申请的前面，JavaScript 会自动提升该申明的顺序，将其提升到调用之前。不过只能提升声明，不能提升赋值的顺序。
对于函数也一样。

  1. JavaScript 会将函数声明和变量声明提升到当前作用域的顶部。
  2. 变量赋值不会提升。
  3. 在脚本的顶部声明函数和变量，这样语法和行为就会相互保持一致。

声明函数

我们有两种声明函数的方式，函数声明和函数表达式

在 JavaScript 中，你也可以向变量中存储函数。存储在变量中的函数叫做函数表达式。

函数表达式和提升

何时使用函数表达式及何时使用函数声明取决于几项内容，你将在下个部分看到几种使用方法。但是，需要注意的一点是提升。
所有函数声明提升和加载后，脚本才会实际地运行。函数表达式不会提升，因为它们涉及变量赋值，只有变量声明会提升。在解析器在脚本中到达该表达式之前，函数表达式不会加载。

作为参数的函数

可以将函数存储在变量中使我们能够轻松地将函数传递到另一个函数中。传递到另一个函数中的函数叫做 回调。假设有个 helloCat() 函数，你希望它返回 “Hello”，后面跟着字符串“meows”，就像 catSays 一样。你可以使 helloCat() 接受回调函数，并传入 catSays，而不用手动操作。
